新建账号分配服务器的策略

策略：预配置n台服务器，新注册账号在n台机器间平均分配；后续新增服务器时，扩大平均分配的机器范围。
基于原因：
  1.平均分配的细化轮询机制，可实现服务器动态调整补给；
  2.不存在服务器浪费问题：即使前期多备一些服务器，也能在开服前几日高峰期过后快速合服，减少闲置；

详细算法：
  1.计算当前注册人数，当前注册人数=max(服务器同步过来的已注册人数, 登录服记录的已注册人数)，即两者取大值；
  2.检查服务器可注册人数（可注册人数：regAvail = 最大注册人数 - 当前注册人数 - 排队人数），当regAvail <= 0，此服务器不用于分配新账号；
  3.计算服务器状态系数 stateRate =(当前在线人数 + 排队人数) / (最大在线人数 + 最大排队人数)
  4.计算服务器注册系数 regRate = (1 - stateRate) * regAvail
  5.取regRate最大值，将此服务器作为新账号的服务器（确保regRate>0)
  6.如果所有regRate==0，则没有可用服务器
